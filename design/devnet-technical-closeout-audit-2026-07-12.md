# Devnet technical closeout audit

> **Unreviewed Codex output.** This report was generated by Codex and has not
> been reviewed or ratified by the project owner.

## Scope

This was a bounded, read-mostly audit for strictly technical work that could be
completed without deciding market sources, product behavior, economics, the
real-ZK architecture, or public-launch policy. It covered the current route
boundary, deployment checks, persistence testing, recovery instructions, and
the live dev-box operational posture. No deployment was performed.

## Completed from the audit

1. **Bound anonymous realtime connections.** Public SSE and v2 WebSocket
   subscriptions previously had no admission cap. Each client held a task and
   broadcast receiver indefinitely, and every block incurred work per client.
   The API now has a shared, configurable 256-connection default, holds permits
   for the full stream lifetime, returns `429` on saturation, and exports active
   and rejected metrics. The authenticated service stream remains separate.
2. **Exercise the deployed v2 stream contract.** The post-deploy gate now runs
   the existing dependency-free WebSocket checker against `/v2/blocks/ws` and
   requires replay, `replay_complete`, and the subsequent live handoff. It also
   inspects the deployed Next.js shell chunks for the `/v2/blocks/ws` client
   path, catching coordinated API/web protocol drift rather than merely
   checking REST health.
3. **Broaden durability coverage.** The parameterized persistence crash harness
   now runs three deterministic seeds by default instead of one: 27 randomized
   crash/recovery cases across nine durability boundaries.
4. **Repair the operator-replacement drill.** The documented manifest URL now
   names `/v1/da/{height}/manifest`, and the private payload request includes
   the required service bearer authentication.

## Important findings deliberately not changed

- **Backups are manual.** The runbook asks for daily and pre-deploy backups,
  but scheduling and off-host retention require an explicit operator storage
  destination and retention policy.
- **Disk exhaustion lacks a dedicated alert.** The dev box was observed around
  72% full during this audit, with substantial Docker images/build cache and
  on-host backups. Warning/critical filesystem alerts are a safe next slice;
  automatic pruning is not safe without deciding rollback and backup retention.
- **Durable account history is append-only.** Fill, equity, and account-event
  tables have no age/row retention. Adding it affects cursor and historical API
  semantics and is a medium tranche, not quick cleanup.
- **Live trading alerts were firing.** `LiveTradingNoFills` and
  `AcceptedOrdersNoFills` were active while the audit ran even though the chain
  and synthetic probe were healthy. That needs a separate evidence-driven
  investigation; changing alert thresholds or forcing fills would be guesswork.
- **The production topology still uses the mock prover.** This is known large
  protocol work and was outside the bounded devnet closeout.

## Deployment note

The privacy boundary changes move the public realtime endpoint from canonical
v1 to sanitized v2 while retaining v1 for authenticated services. API and web
should therefore be promoted together with `deploy-all`; a narrow deployment
of only one side can create a protocol mismatch. Build remains local—the dev
box should only load the prepared images.
