## Purpose

The **matching-scenarios** crate generates synthetic prediction market order books for testing and benchmarking. It provides reproducible scenario generation with configurable complexity.

## ScenarioConfig Fields

**Market Configuration:**
- `num_markets` тАФ number of binary markets
- `market_group_probability` тАФ rate of 3-4 market mutually exclusive groups

**Order Configuration:**
- `num_orders` тАФ total orders to generate (all single-market)
- `order_size_min/max` тАФ quantity range
- `order_size_power` тАФ uniform (`1`) or small-order-heavy (`>1`) size shape
- `retail_buy_probability` тАФ global buy/sell-side imbalance
- `seed` тАФ ChaCha8 seed for reproducibility

**Liquidity Configuration:**
- `liquidity_scarcity` (0.0-1.0) тАФ lower = scarcer = tighter matching
- `hot_market_fraction` тАФ markets receiving extra demand
- `hot_order_probability` тАФ concentration of flow in hot markets
- `market_activity_power` тАФ optional long-tailed market popularity
- `liquidity_depth_levels` тАФ background-book levels per outcome and side
- `liquidity_dispersion` тАФ log-normal cross-market depth heterogeneity

**Market Maker Configuration:**
- `num_mms` тАФ number of MM constraints
- `mm_budget_min/max` тАФ budget range in dollars
- `mm_spread_bps` тАФ spread in basis points
- `mm_capacity_multiplier` тАФ quote capacity relative to budget
- `mm_market_coverage_fraction/max` тАФ breadth of each MM's quote set
- `mm_quote_style` тАФ synthetic ladder or live-shaped one-shot flash quotes

## Generation Process

1. Create N binary markets with latent fair prices
2. Group a configurable share into mutually exclusive sets
3. Add configurable, optionally heterogeneous background-book depth
4. Generate single-market orders (buy/sell YES/NO)
5. Add partially overlapping MM ladders or live-shaped flash quotes and capital constraints
6. Shuffle orders to avoid order-dependent solver behavior

## Reproducibility

All scenarios use seeded RNGs (ChaCha8Rng). Same seed = same Problem.

```rust
let config = ScenarioConfig { seed: 12345, ..ScenarioConfig::medium() };
let p1 = generate_scenario(config.clone());
let p2 = generate_scenario(config);
assert_eq!(p1.orders.len(), p2.orders.len()); // Deterministic
```

## Module Structure

| Module | Purpose |
|--------|---------|
| `scenario.rs` | Primary generator with `ScenarioConfig` |
| `random.rs` | Simpler `RandomConfig` for basic testing |
| `flash_liquidity.rs` | Two-sided MM bid/ask ladders under one shared budget |

The paper/solver benchmark profiles live in
`benchmarks/solver/protocol-v2.json`; v1 is retained as dated evidence. Scenario
size fields are whole shares and are converted to protocol quantity units at
generation time. These are stylized structural stress tests, not a substitute
for a calibrated real-order replay dataset.
